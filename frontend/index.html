Request Flow & Data Pipeline
1
Frontend Upload: User selects image + enters name
POST /upload multipart/form-data
2
Image Processing: Flask backend resizes to 800x800, converts to JPEG
PIL.Image.thumbnail() → io.BytesIO()
3
S3 Storage: Image uploaded with UUID as key
boto3.s3.put_object(ContentType='image/jpeg')
4
SageMaker Inference: Real-time object detection via HTTP endpoint
runtime.invoke_endpoint(ContentType='application/x-image')
5
Result Parsing: Extract classes, scores, filter confidence > 30%
COCO class mapping, top 10 detections
6
Bedrock Generation: LLM creates natural language scene description
bedrock.invoke_model('amazon.nova-micro-v1:0')
7
DynamoDB Logging: Each detection stored as separate item
table.put_item(image_id, timestamp, object, confidence)
8
Response: JSON payload with detections + scene description
200 OK application/json
Model Configuration
Model: SSD MobileNet v2 FPN-Lite 320x320 Framework: TensorFlow 2.8.0 Dataset: COCO (80 object classes) Container: 763104351884.dkr.ecr.eu-west-1.amazonaws.com/tensorflow-inference:2.8.0-cpu Endpoint: jumpstart-dft-mobilenet-v2-fpnlite-20260403-130823 Instance Type: ml.m5.large (2 vCPU, 8GB RAM) Inference Time: ~200-400ms average
DynamoDB Schema
Table: detection-results Partition Key: image_id (String) Sort Key: timestamp (String, ISO 8601) Attributes: - object_name (String) - confidence (Number, 0-100) - inference_ms (Number) - uploader_name (String) - scene_description (String, from Bedrock) - source (String, default: 'web-upload')
CI/CD Pipeline (GitHub Actions)
Trigger: Push to main branch Runner: ubuntu-latest Steps: 1. SSH to EC2 via appleboy/ssh-action@v1.0.3 2. Git reset --hard origin/main 3. pip install -r requirements.txt --break-system-packages 4. pkill gunicorn; start.sh Secrets: - EC2_KEY (RSA PEM private key) - EC2_HOST (54.74.221.183)
