# Object Detection App — CS5024

An end-to-end object detection web application built on AWS, developed as part of **CS5024: Theory and Practice of Advanced AI Ecosystems** at the University of Limerick.

Users upload an image through the browser, the backend runs inference on a SageMaker-hosted model, Amazon Bedrock generates a natural-language description of the scene, and all results are stored in DynamoDB for later analytics.

---

## Features

- **Object detection** using MobileNet v2 FPN-Lite (COCO, 80 classes) deployed on Amazon SageMaker
- **Scene description** generated by Amazon Bedrock (Nova Micro) based on detected objects
- **Analytics dashboard** with detection history, stats, and per-object confidence, backed by DynamoDB
- **Image storage** in Amazon S3
- **CI/CD** with GitHub Actions — push to `main` automatically redeploys to EC2
- **Dark-themed single-page frontend** with drag-and-drop upload, live detection pills, and tabbed views

### AWS Services

| Service      | Purpose                                   |
|--------------|-------------------------------------------|
| EC2          | Hosts the Flask + Gunicorn backend        |
| S3           | Stores uploaded images                    |
| SageMaker    | Runs the `tensorflowmodel` endpoint       |
| DynamoDB     | `detection-results` table for history     |
| Bedrock      | Nova Micro model for scene descriptions   |
| IAM          | Role-based access between services        |

## API Endpoints

| Method | Endpoint      | Description                                                |
|--------|---------------|------------------------------------------------------------|
| GET    | `/`           | Serves the frontend                                        |
| GET    | `/health`     | Health check — returns `{"status": "healthy"}`             |
| POST   | `/upload`     | Uploads an image, runs detection, returns results          |
| GET    | `/analytics`  | Returns all detection records from DynamoDB                |

### `POST /upload`

**Request** (`multipart/form-data`):
- `file` — image file (JPEG/PNG)
- `uploader_name` — name of the person uploading

**Response**:
```json
{
  "status": "success",
  "image_id": "uuid",
  "s3_url": "https://object-detection-uploads-v1.s3.amazonaws.com/<image_id>",
  "detections": [
    { "name": "person", "confidence": 92.4 },
    { "name": "dog", "confidence": 87.1 }
  ],
  "inference_ms": 412.7,
  "scene_description": "A person is walking their dog outdoors."
}
```

Detections below **30% confidence** are filtered out and the top 10 are returned.

## Configuration

The backend reads its AWS configuration from constants in `backend/app.py`:

```python
BUCKET         = "object-detection-uploads-v1"
ENDPOINT       = "tensorflowmodel"
REGION         = "eu-west-1"
DYNAMODB_TABLE = "detection-results"
```

Bedrock runs in `us-east-1` (Nova Micro availability).

### DynamoDB schema (`detection-results`)

| Attribute           | Type   | Notes                                   |
|---------------------|--------|-----------------------------------------|
| `image_id`          | String | Partition key (UUID per upload)         |
| `timestamp`         | String | Sort key (ISO 8601 UTC)                 |
| `object_name`       | String | Detected class name                     |
| `confidence`        | String | Confidence percentage                   |
| `inference_ms`      | String | Inference latency in milliseconds       |
| `uploader_name`     | String | Provided by the user                    |
| `scene_description` | String | Bedrock-generated one-liner             |
| `source`            | String | Always `web-upload`                     |

## Local Development

1. Clone the repo:
   ```bash
   git clone https://github.com/vivek19398/object-detection-appv1.git
   cd object-detection-appv1/backend
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Configure AWS credentials (the instance needs access to S3, SageMaker, DynamoDB, and Bedrock):
   ```bash
   aws configure
   ```

4. Run the Flask app:
   ```bash
   python app.py
   ```

   The app will be available at `http://localhost:8000`.

> **Note:** The frontend has the EC2 IP hardcoded in `API_URL`. For local testing, change it to `http://localhost:8000`.

## Deployment

Deployment is fully automated. Any push to the `main` branch triggers `.github/workflows/deploy.yml`, which:

1. SSHes into the EC2 instance
2. Pulls the latest code
3. Installs dependencies
4. Restarts Gunicorn via `start.sh`
5. Verifies the `/health` endpoint

### Required GitHub secrets

- `EC2_HOST` — public IP or hostname of the EC2 instance
- `EC2_KEY` — private SSH key for the `ubuntu` user

---

## IAM Permissions

The EC2 instance role needs:

- `s3:PutObject`, `s3:GetObject` on the uploads bucket
- `sagemaker:InvokeEndpoint` on the `tensorflowmodel` endpoint
- `dynamodb:PutItem`, `dynamodb:Scan` on the `detection-results` table
- `bedrock:InvokeModel` on `amazon.nova-micro-v1:0`
